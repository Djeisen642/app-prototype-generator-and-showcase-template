# Research Docs

When a prototype is backed by research, keep it here as `docs/research/{topic}.md`,
one file per topic. The detail-page brief (overview, problem, features, voice) is the
distilled product of this work — the research doc is where the evidence and reasoning live.

Hold every doc to the **"Pulitzer light"** standard described in `CLAUDE.md`
(and summarized in `AGENTS.md`):

1. Primary sources, read in full — not search snippets or blog summaries.
2. Attribute every key claim — author, year, sample size, source.
3. Report effect sizes and confidence intervals, not just direction.
4. State confidence honestly — flag thin/unverified claims as such.
5. The change-my-mind test — call out where a source contradicts the prior.
6. Separate "what's known" from "what I'm inferring."

End each doc with a **Sources** section and a one-paragraph honest note on where the
evidence is strong vs. weak/contested.
